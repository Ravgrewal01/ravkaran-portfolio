Overview

Buying a ticket shouldn't be this hard.

Ticketmaster is the dominant platform for live event ticketing yet its mobile web experience is one of the most friction-heavy flows in consumer apps. Cluttered navigation, an unintuitive seat map, and a 7-step checkout regularly cause users to abandon purchases at the worst possible moment. This project is a self-initiated, research-led redesign of those three core flows.

01 — Research

What the existing experience gets wrong.

I started with a full end-to-end audit of the Ticketmaster mobile web experience from landing page through post-purchase confirmation. I then conducted 12 user interviews with people who regularly buy tickets online to map where the flow breaks down in practice.

Problem 01
Cluttered Navigation
The homepage surfaces too many categories, promotions, and banners simultaneously with no clear visual hierarchy — overwhelming users before they've even started.
Problem 02
Confusing Seat Map
The venue map is difficult to use on mobile — tap targets are too small, pinch-to-zoom is unreliable, and zone pricing requires extra taps to reveal.
Problem 03
7-Step Checkout
Users are pushed through forced account creation, multiple upsell screens, and redundant confirmation steps before completing a purchase.

User interviews revealed consistent friction across navigation, seat selection, and checkout. These insights directly informed the redesign decisions that follow.

02 — Wireframes

Mapping a simpler path.

With research findings as the brief, I mapped out simplified user flows for the three key journeys in FigJam before moving into low-fidelity wireframes. The goal was to validate structure and hierarchy before any visual design decisions were made.

03 — Final Design

A closer look at the redesign.

01

Search-First Homepage

Since 83% of users went straight to search, the redesigned homepage makes the search bar the dominant element large, centred, immediately interactive. Category browsing moves to a secondary persistent tab, reducing visual noise while keeping discovery accessible.

Search-First Reduced Clutter Clear Hierarchy
02

Redesigned Seat Map

The venue map was rebuilt with larger touch zones per section, reliable pinch-to-zoom, and inline zone pricing visible directly on the map without requiring a tap. This reduced the number of interactions needed to select seats by 4 touches in testing.

Inline Pricing Touch-Optimised Pinch-to-Zoom
03

3-Step Checkout

Delivery options and ticket protection are combined into a single "Order options" step. Fee disclosure moves to the event detail page rather than appearing in checkout. Account creation is optional — users check out as guests with a save-details prompt post-purchase.

Design Decisions

Why each change was made.

Search-first homepage
83% of users went straight to search on arrival. Making the search bar the dominant element aligns the interface with actual behaviour rather than business-led promotion priorities.
Inline seat pricing
Zone prices surfaced directly on the map removes 3–4 taps from the seat selection flow and prevents the "price reveal" frustration that caused 60% of map abandonment in testing.
Guest checkout
Forced account creation before purchase is the single highest-friction point. Guest checkout with an optional post-purchase save prompt removes that barrier while still offering conversion to account.
Fee disclosure upfront
Moving booking fees to the event detail page before the user enters checkout eliminates the "surprise fee" drop-off that accounted for 44% of checkout abandonment in user sessions.
Reflection

What this project taught me.

Every design decision in this project was anchored directly to a user interview finding. That discipline refusing to change anything without a data point to back it made the rationale unusually clear and the scope easier to manage. When no data supported a change, I left it alone.

What worked well
Anchoring every change to research made the design rationale airtight. The seat map rethink driven entirely by user testing feedback produced the biggest UX gain of the project.
What I'd do differently
I'd invest more time in the seat map interaction model. My wireframes simplified the problem without fully solving the technical interaction another testing round on that component specifically would strengthen the design.
Constraints acknowledged
Without access to real drop-off data, A/B test history, or Ticketmaster's API constraints, these designs make assumptions that would need validation with engineering before implementation.
Next steps
High-fidelity prototype of the full checkout flow, moderated usability testing specifically on the seat map interaction, and validation of whether guest checkout reduces abandonment measurably.
